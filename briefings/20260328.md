2. 本次新增 `X-Claude-Code-Session-Id` 请求头，官方明确说明代理层现在可以不解析请求体，就按 session 聚合 API 请求。
3. 同版还补了多项工程侧关键修复：将 `.jj` 与 `.sl` 纳入 VCS 元数据目录排除列表，避免 Grep 与文件自动补全深入 Jujutsu / Sapling 元数据；修复旧 session 的 `--resume` 失败、`--bare` 模式丢失 MCP tools、带条件 skills/rules 时无法读写项目根外文件等问题；同时 Read 工具改为更紧凑的行号格式并去重未变化重读，降低 token 开销。

2. 为什么值得工程团队关注：这次更新虽然不是“新能力大版本”，但对企业内接入和长期运行更关键。会话级 header 让代理、网关、审计日志和成本归因更容易按一次 agent 任务聚合；对 `.jj` / `.sl` 的排除则说明 Claude Code 开始更认真处理非 Git 工作流；而 MCP、恢复、根外文件访问与 token 开销优化，直接影响团队把 agent 接进真实开发环境后的稳定性与运营成本。

3. 最可能影响的环节：开发 / 测试 / 流程。

【工程意义】
- 对开发：使用 Jujutsu、Sapling、home 目录规则文件或长会话恢复的团队，日常可用性会更高，减少“不是代码错、而是工具状态坏了”的摩擦。
- 对测试：需要补回归验证 `--resume`、`--bare`+MCP、项目根外规则文件访问，以及长会话下 Read 工具与 `/feedback` 等高 token/大 transcript 场景。

### 二、Gemini CLI 在北京时间 3 月 28 日同时发布稳定版与预览版补丁，沙箱默认网络权限从关闭改为开启

1. Google 官方 GitHub Releases `v0.36.0-preview.6` 与 `v0.35.3` 的 `published_at` 分别为 `2026-03-28T03:20:18Z`、`2026-03-28T03:24:15Z`，均落在北京时间 2026-03-28，当日稳定版与预览版同步发补丁，时间边界成立。
2. 两个 release notes 都明确说明它们是对同一修复的 cherry-pick；对应官方 PR `#24055` 与提交 `765fb67` 标题均为 **Enable network access in sandbox configuration**。
3. 该提交把 Gemini CLI 沙箱配置里的 `networkAccess` 默认值从 `false` 改为 `true`，并同步更新测试，覆盖 Docker、LXC、sandbox-exec、Podman、runsc 等多种沙箱运行方式；也就是说，如果团队此前依赖默认值，沙箱现在会默认允许联网，除非显式设为禁用。

2. 为什么值得工程团队关注：这不是普通的 patch 修修补补，而是默认安全与可用性边界的变化。对需要在沙箱里安装依赖、访问私有包仓、抓文档或跑联网生成任务的团队，这会减少失败与额外配置；但对把“沙箱默认离线”当作安全前提的团队，这等于一条必须显式复核的策略变更。

3. 最可能影响的环节：开发 / 测试 / 流程 / 安全选型。

【工程意义】
- 对开发：依赖下载、代码生成、文档抓取、联网工具调用这类任务在沙箱里更可能直接跑通，减少为了联网额外调参的摩擦。
- 对测试：原来基于“默认无网络”写的回归、快照或安全测试可能出现行为漂移；同时也需要补“显式禁网时是否仍然严格禁用”的反向验证。
- 对流程：平台、安全和 DevEx 团队不能再把默认行为当成隐含策略，需要在团队模板、starter config 和审计规则里显式写清楚 `networkAccess` 预期。

【建议动作】
- 立即可做：在团队的 Gemini CLI 配置模板、示例仓库和 CI/本地 bootstrap 文档中搜索是否存在“默认离线沙箱”的隐含假设；若有，显式写入 `networkAccess: false`。
- 本周可验证：分别验证两类路径——一类是需要联网的安装/抓取/生成任务是否因此减少失败；另一类是高约束仓库在显式禁网后是否仍满足最小权限与可重复执行要求。
- 暂时观察：稳定版与预览版同日一起修这条行为，说明变更优先级较高；在严格隔离或合规环境里，先锁版本并完成基线复核，再决定是否放开默认联网。

【来源】
- GitHub Releases：google-gemini/gemini-cli `v0.35.3`
- GitHub Releases：google-gemini/gemini-cli `v0.36.0-preview.6`
- 官方仓库 PR：`#24055` Enable network access in sandbox configuration
