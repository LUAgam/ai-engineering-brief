## 生成式AI / 工程工具链

### 一、OpenAI Agents Python v0.13.0 扩展 MCP 能力，并把默认 Realtime WebSocket 模型切到 `gpt-realtime-1.5`

1. 官方 GitHub Release `publishedAt` 为 `2026-03-23T06:19:55Z`，换算为北京时间是 `2026-03-23 14:19:55`，时间边界成立。
2. 该版本新增 `MCPServer.list_resources()`、`list_resource_templates()`、`read_resource()`，并让 `MCPServerStreamableHttp` 暴露 `session_id`，可用于 streamable HTTP 会话在重连或无状态 worker 场景下恢复。
3. release 同时把默认 Realtime WebSocket 模型改为 `gpt-realtime-1.5`，并修复多项运行时边界问题，包括 `SQLAlchemySession` 首次并发写入、`remove_all_tools()` 未清理 MCP/推理项，以及函数工具批执行竞态。

为什么值得工程团队关注：这不是单纯 patch 版修小 bug，而是在同时推进两条工程主线：**MCP server 资源访问能力更完整**，以及 **Realtime/Session 运行时默认值和稳定性进一步收敛**。如果团队在做 Agent 平台层封装、MCP 接入或语音/实时交互链路，这类更新会直接影响适配层代码与回归集。

最可能影响的环节：开发 / 测试 / 流程。

【工程意义】
- 对开发：如果你们自己封装了 `MCPServer` 抽象，资源枚举和读取能力现在可以直接走官方接口，不必继续维护私有扩展。
- 对测试：默认 Realtime 模型切换意味着即便你们“不改配置”，首轮行为也可能发生变化；需要把延迟、输出风格、工具调用连续性重新纳入基线。
- 对流程：`session_id` 暴露后，更适合把 streamable HTTP MCP 接到弹性 worker 或可恢复会话场景，但也要求补会话恢复与幂等检查。

【建议动作】
- 立即可做：检查内部 SDK 封装或样板代码是否显式假设旧默认 Realtime 模型；同时评估是否要接入新的 MCP 资源读取接口替代自建实现。
- 本周可验证：补一轮回归，重点覆盖 MCP 资源枚举/读取、会话恢复、并发 session 写入，以及工具批执行顺序稳定性。
- 暂时观察：如果你们对 Realtime 输出表现或 provider 兼容层高度敏感，可先固定模型版本，待新默认值通过真实流量验证后再放开。

【来源】
- OpenAI 官方 GitHub Release：`openai/openai-agents-python` `v0.13.0`

### 二、OpenClaw `2026.3.22` 正式发布，集中调整插件 SDK、Browser MCP 和多项安全默认项

1. 官方 GitHub Release `publishedAt` 为 `2026-03-23T11:11:22Z`，换算为北京时间是 `2026-03-23 19:11:22`，时间边界成立。
2. 该版本包含一组明确的 breaking changes：公共插件 SDK 统一到 `openclaw/plugin-sdk/*`，移除 `openclaw/extension-api`；插件安装优先走 ClawHub；旧环境变量别名和 `.moltbot` 状态目录兼容逻辑被移除。
3. Browser/Chrome MCP 链路也发生迁移：旧 Chrome extension relay 路径和相关配置被移除，要求迁移到 `existing-session` / `user` 或继续使用 raw CDP；release 同时增加多项安全加固，例如收紧 webhook 预认证窗口、限制 build-tool/JVM 注入与部分环境变量利用路径。

为什么值得工程团队关注：这类 release 影响的不是“模型效果”，而是 **Agent 平台可维护性、浏览器自动化接入方式、插件供应链和默认安全边界**。如果团队把 OpenClaw 用作编码 agent、浏览器代理或多渠道机器人底座，这次升级会直接改变迁移脚本、配置模板和测试矩阵。

最可能影响的环节：开发 / 测试 / 流程 / 选型。

【工程意义】
- 对开发：插件作者和平台维护者需要尽快迁移 SDK 引用路径，并检查 Browser MCP 配置是否仍基于旧 relay 方式。
- 对测试：不能只跑 smoke test；需要补插件安装、Browser MCP、配置迁移、webhook 入口和 exec 审批/环境注入的专项回归。
- 对流程：ClawHub 优先安装与旧兼容项移除，意味着团队要重新梳理插件来源、版本锁定、升级 SOP 和安全审查清单。

【建议动作】
- 立即可做：全文搜索 `openclaw/extension-api`、旧 Chrome relay 配置、`CLAWDBOT_*` / `MOLTBOT_*` 旧环境变量，以及 `.moltbot` 路径依赖。
- 本周可验证：在隔离环境跑一轮升级演练，重点验证 `openclaw doctor --fix`、Browser 任务、插件安装来源、webhook 行为和安全回归。
- 暂时观察：如果你们当前依赖大量自定义插件或浏览器自动化流程，建议先冻结生产升级，等迁移项和回归基线补齐后再切换。

【来源】
- OpenClaw 官方 GitHub Release：`openclaw/openclaw` `v2026.3.22`
