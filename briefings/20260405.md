## 工程工具链

### 一、opencode 在北京时间 4 月 5 日连续发布 v1.3.14 / v1.3.15，重点补 review 恢复链路、企业托管配置与 npm 安装稳定性

1. `anomalyco/opencode` 官方 GitHub Releases 显示，`v1.3.14` 和 `v1.3.15` 的 `published_at` 分别为 `2026-04-04T18:32:40Z`、`2026-04-04T20:54:43Z`，折算北京时间为 `2026-04-05 02:32:40`、`2026-04-05 04:54:43`，时间边界成立。  
2. `v1.3.14` 的核心改动集中在“恢复”和“治理”：恢复基于 git 的 review mode（含 uncommitted diff 与 branch diff）；修复 revert chain 恢复早期消息时快照状态错误；新增 macOS managed preferences，支持企业用 MDM 强制下发配置；同时修复 OpenAI-compatible provider 工具调用后 session 卡死。  
3. 随后发布的 `v1.3.15` 继续补工程落地短板：专门修复 npm 安装时因 Arborist 命中编译后二进制 `node-gyp` 路径而失败的问题，属于典型“能否在真实项目依赖树里装起来”的稳定性补丁。

为什么值得工程团队关注：这两版说明编码 agent 的发布重点已经从“新交互”转向“回看 diff 是否可靠、企业配置能否集中管控、安装链路会不会在 node-gyp 项目里炸掉”。这些问题不解决，团队很难把 agent 放进日常评审和多项目 rollout。

最可能影响的环节：开发 / 测试 / 流程 / 企业管理。

【工程意义】
- 对开发：恢复 uncommitted diff / branch diff review 后，agent 参与代码审查时更接近真实 Git 工作流；修复 OpenAI-compatible provider 会话卡死，也更利于多模型接入。
- 对测试：应重点补回归用例覆盖 review mode、revert chain 快照恢复、npm + node-gyp 项目安装、Windows / macOS 上 SDK / TUI 启停与配置注入。
- 对流程：macOS managed preferences 让平台团队可以把模型、策略和默认行为纳入 MDM 下发，而不是靠开发者本地手改。

### 二、Anthropic 的 claude-code-action 在北京时间 4 月 5 日发布 v1.0.89，把 GitHub Actions 里的安全边界、重试语义与仓库边角问题继续收紧

1. `anthropics/claude-code-action` 官方 GitHub Release `v1.0.89` 的 `published_at` 为 `2026-04-05T15:35:26Z`，折算北京时间为 `2026-04-05 23:35:26`，时间边界成立。  
2. 这版最关键的修复集中在 CI 稳定性与安全边界：当没有真正拿到 token 时跳过 revocation；从 Claude session 中剥离 OIDC token request 相关环境变量；对 non-retryable error 不再盲目重试；同时改进日志洁净度并固定 bun runtime 配置。  
3. 同版还修了多类真实仓库更容易踩坑的边角：`bun install --production` 后恢复 `rg` 可执行位；支持带 `#` 的分支名用于 PR checkout / base restore；修复含 `.gitmodules` 仓库在 `restoreConfigFromBase` 阶段卡住；在 security restore 前先把 PR 的 `.claude/` 快照到 `.claude-pr/`；并补充 `include_comments_by_actor` / `exclude_comments_by_actor` 的文档。

为什么值得工程团队关注：如果你们把 Claude Code 放进 PR review、issue triage 或 workflow_run 自动化，真正影响可用性的往往不是模型效果，而是 token 处理、重试语义、repo 边角兼容和日志安全卫生。`v1.0.89` 这轮修复基本都落在这条线上。

最可能影响的环节：开发 / 测试 / 流程 / 安全治理 / CI。

【工程意义】
- 对开发：带 `#` 的分支名、含 `.gitmodules` 的仓库和 `rg` 可执行位恢复，都是多仓 / 子模块 / 复杂 runner 环境里常见但容易漏测的问题。
- 对测试：应补 workflow_run 场景、non-retryable error、OIDC 环境变量隔离、`.claude/` 快照恢复以及 `include/exclude_comments_by_actor` 输入组合测试。
- 对流程：OIDC 请求变量剥离、无 token 不 revocation、减少无意义重试，有助于把 GitHub Action review 流程做得更少噪音、更可审计。

【建议动作】
- 立即可做：若团队已在用 `claude-code-action`，先把版本锁到 `v1.0.89` 或在影子 workflow 中灰度验证，并检查日志里是否仍暴露 OIDC 相关环境变量。
- 本周可验证：新增一组 CI 回归，覆盖带 `#` 分支名、含 `.gitmodules` 的 PR、non-retryable error、`bun install --production` 后 `rg` 仍可执行等场景。
- 暂时观察：继续观察 Anthropic 是否在后续版本把 comment actor 过滤、security restore 与 workflow_run 示例进一步标准化。

- GitHub Releases：`anthropics/claude-code-action v1.0.89`
